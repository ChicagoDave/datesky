## Key Decisions

### Execution-Time Decisions Resolved This Session

**(a) Cookie rename deferred to Phase 3**
The `src/lib/session.ts` session cookie name (`datesky-session`) was kept unchanged. Renaming it requires forced re-login for all active users. This is bundled with the Phase 3 OAuth client cutover to produce a single re-login event rather than two.

**(b) DB filename kept with comment**
`data/datesky.db` retained as-is with an explanatory comment in `src/lib/db/index.ts`. Renaming deferred to Phase 4 to avoid a one-off VPS copy/symlink step; the rename will be bundled with the Phase 4 infra migration.

**(c) GitHub URLs left for Phase 5**
`package.json` `repository.url` / `homepage` / `bugs.url` and the GitHub link in `src/app/about/page.tsx` are left pointing at the current repo. Updating them now would 404 until the GitHub repo is actually renamed. Phase 5 handles all GitHub-URL work simultaneously with the repo rename.
