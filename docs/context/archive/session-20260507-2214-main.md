## Key Decisions

### Phase 1 Execution-Time Decisions

**(a) Cookie rename deferred to Phase 3**
The `src/lib/session.ts` session cookie name (`datesky-session`) was kept unchanged. Renaming it requires forced re-login for all active users. This is bundled with the Phase 3 OAuth client cutover to produce a single re-login event rather than two.

**(b) DB filename kept with comment**
`data/datesky.db` retained as-is with an explanatory comment in `src/lib/db/index.ts`. Renaming deferred to Phase 4 to avoid a one-off VPS copy/symlink step; the rename will be bundled with the Phase 4 infra migration.

**(c) GitHub URLs left for Phase 5**
`package.json` `repository.url` / `homepage` / `bugs.url` and the GitHub link in `src/app/about/page.tsx` are left pointing at the current repo. Updating them now would 404 until the GitHub repo is actually renamed. Phase 5 handles all GitHub-URL work simultaneously with the repo rename.

### Phase 2 Execution-Time Decisions

**(1b) Real-path test: manual with operator credentials (not containerized)**
The real-path test for dual-publish (`scripts/test-dual-publish.ts`) requires a live PDS connection and therefore a real Bluesky test account + app password. Setting up a containerized PDS was judged out of scope for this session. The script is operator-invocable and documents its own requirements in the header. A containerized harness remains an option for a future session.

**(2) Legacy mirror write failure mode: log + return success**
If the best-effort legacy mirror write to `app.datesky.profile` fails, the PUT handler logs the error and returns HTTP 200 to the client. The canonical write to `app.nomare.profile` is the user-facing contract; the legacy write is infrastructure plumbing. Treating its failure as a 500 would surface an implementation detail to users. Recorded in ADR-0003.

**(3) Legacy lexicon flagged via description field**
`lexicons/app/datesky/profile.json` has no valid comment syntax (JSON). The `description` field was updated to read "LEGACY — superseded by app.nomare.profile via dual-publish (ADR-0003). Do not extend." This is the closest available equivalent to a deprecation comment in a JSON lexicon file.
