## Key Decisions

### 1. Spaces normalize to hyphen only (no underscore)

User chose hyphen as the sole whitespace replacement character. Underscore was not added to the allowed character set. This keeps tag slugs consistent with common URL conventions and avoids two valid representations of the same semantic tag.

### 2. Normalize everywhere, not just at the gate

All four write sites (client input, server PUT, indexer upsert, flag recompute) normalize before validating. A prior approach of normalizing only at the API boundary would have left the indexer still flagging Jetstream-sourced space tags. Uniform application means the `--apply` script is the only backfill needed.

### 3. Backfill via recompute, not a migration

The existing `recomputeInvalidTagFlag --apply` script reads `raw_record` through the new normalize-first rules and resets `has_invalid_tags` accordingly. No schema migration needed. Jetstream events will reconcile stored tag strings for active users over time; `tag-cleanup.ts` (delete-only) is left as-is since normalize-and-rewrite is a future polish item.
